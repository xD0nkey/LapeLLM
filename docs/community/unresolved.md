# Unresolved Community Notes

This file holds community-sourced hints that are too vague, too incomplete, or too poorly scoped to file under a specific topic (`gui.md`, `minimap.md`, `debugging.md`, `api-notes.md`) yet. A note belongs here when it does not yet have enough identifiable content (a clear symbol, a clear topic, a clear claim) to be usefully categorized — not simply because its confidence is low. A note can be filed under a specific topic file with `Unverified` or even `Contradictory` confidence as long as it has a clear subject; see `docs/community/minimap.md` for an example of an unverified-but-categorized note.

See `docs/community/index.md` for the full policy, and `docs/community/intake-template.md` for the entry format. Use the same template here as in the topic files.

Workflow for this file:

1. A vague or under-specified note is logged here using the intake template, with "What is unknown" and "What would confirm this" filled in as completely as possible even though the topic itself is unclear.
2. If later context, a follow-up message, or an example clarifies the topic, move the entry to the appropriate topic file (`gui.md`, `minimap.md`, `debugging.md`, or `api-notes.md`) rather than duplicating it.
3. If a note is confirmed outright, move the confirmed information into the correct primary `docs/` topic file instead, and remove it from here.
