# Community Note Intake Template

Copy this template when adding a new community-sourced note to `docs/community/`. Fill in every section. Do not skip "What is unknown" or "Unsafe assumptions" — they are as important as the parts that look confirmed.

## Raw snippet

```text
Paste the original chat snippet here.
```

## Topic

Example: GUI overlay, minimap, debugging, mouse input, color detection, pathfinding, WaspLib API.

## Mentioned symbols

List exact symbols, names, functions, properties, methods, constants, or files mentioned.

## Plain English interpretation

Explain what the snippet appears to mean without adding unsupported details.

## Confidence

Choose one:

- Confirmed
- Likely
- Unverified
- Contradictory
- Deprecated

## Source type

Choose one or more:

- Community discussion
- Local example
- Local documentation
- WaspLib source
- WaspLib documentation
- User provided example

## Evidence

Link or reference the local file, external documentation page, source file, or example that supports the note.

## What is known

List only what is directly supported.

## What is unknown

List missing context, missing function signatures, missing required imports, missing object ownership, missing runtime assumptions, or unclear behavior.

## What would confirm this

List the example, source file, documentation page, or working script needed to confirm the note.

## Safe usage guidance

Explain how an AI agent may use this note without overclaiming.

## Unsafe assumptions

List assumptions that must not be made.
